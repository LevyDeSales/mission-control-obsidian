Criação de Bases de Conhecimento (CRIACAO_BASES)

Neste vault, o termo "Base" refere-se tecnicamente aos arquivos de definição de visualização (.base) localizados em /Modelos/Bases. Eles são o motor que gera as tabelas dinâmicas nas notas de Categoria.

1. O que é um Arquivo Base

Um arquivo .base é um arquivo de configuração escrito em YAML que instrui o plugin Bases sobre como consultar, filtrar e exibir notas.

  • Não é uma nota: Não contém texto corrido.
  • Localização: /Modelos/Bases.
  • Extensão: .base.

2. Estrutura de um Arquivo Base

A anatomia de um arquivo base consiste em três seções principais:

1
filters:      # Define QUAIS notas aparecem
2
  and:
3
    - categorias.contains(link("Livros"))
4
properties:   # Define QUAIS colunas aparecem e seus nomes
5
  nota.autor:
6
    displayNome: Autor
7
  file.name:
8
    displayNome: Título
9
views:        # Define AS TABELAS (Views) disponíveis
10
  - type: table
11
    name: Todos os Livros
12
    order:
13
      - file.name
14
      - autor
15
    sort:
16
      - property: file.name
17
        direction: ASC

3. Como Criar uma Nova Base

Sempre que você criar uma nova Categoria (ex: "Animes"), você precisará de uma Base correspondente.

Passo a Passo

  1. Navegue até /Modelos/Bases.
  2. Crie um novo arquivo chamado Animes.base.
  3. Defina o Filtro (Filter):
    • A regra mais comum é filtrar pela categoria: categorias.contains(link("Animes")).
  4. Defina as Propriedades (Properties):
    • Mapeie os campos do frontmatter (ex: nota.estúdio) para nomes amigáveis (displayNome: Estúdio).
    • file.name é obrigatório para mostrar o link da nota.
  5. Defina as Views:
    • Crie pelo menos uma view padrão.
    • Defina a ordem das colunas (order) e a ordenação das linhas (sort).

4. Sintaxe de Filtros

O sistema suporta lógica booleana simples:

  • and: Todas as condições devem ser verdadeiras.
  • or: Pelo menos uma condição deve ser verdadeira.
  • !: Negação (ex: !file.name.contains("Modelo")).

Exemplo complexo:

1
filters:
2
  and:
3
    - categorias.contains(link("Projetos"))
4
    - '!status.contains("Arquivado")'

5. Integração com Categorias

Depois de criar o arquivo .base, você deve "ativá-lo" na nota de Categoria.

  1. Vá para a nota da categoria (ex: /Categorias/Animes.md).
  2. Adicione o transclude do arquivo base: 
    1
    ![[Animes.base]]
  3. O sistema renderizará automaticamente a tabela interativa.

6. Manutenção e Erros Comuns

  • Erro: A tabela está vazia.
    • Causa: O filtro está incorreto ou nenhuma nota tem a categoria [[Animes]] no frontmatter.
  • Erro: Uma coluna está vazia.
    • Causa: O nome da propriedade em properties não corresponde exatamente ao nome no frontmatter das notas (case-sensitive).
  • Modificação: Ao adicionar uma nova coluna em uma Base, ela aparecerá retroativamente para todas as notas que tiverem aquele campo preenchido.
