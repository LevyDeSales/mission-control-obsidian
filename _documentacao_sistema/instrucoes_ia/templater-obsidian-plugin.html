1. Visão Geral Técnica

1.1 Arquitetura e Componentes

O Templater é um motor de templates para Obsidian construído sobre o Eta (um motor de template JS embutido). Ele intercepta a criação de notas ou comandos manuais para processar blocos de código delimitados por tags específicas antes de renderizar o texto final.

  • Core: Estende a classe Plugin da API do Obsidian (obsidian.d.ts).
  • Engine: Utiliza Eta para parsing e execução de JavaScript dentro de delimitadores <% ... %>.
  • Escopo Global (tp): Durante a execução, o plugin injeta um objeto global tp que expõe módulos internos e a API do Obsidian.

1.2 Fluxo de Processamento

  1. Trigger: Evento de criação de arquivo ou inserção manual de template.
  2. Contexto: O plugin instancia o objeto tp com o contexto atual (arquivo ativo, seleção, cursor).
  3. Parsing: O motor Eta varre o texto em busca de tags:
    • <% ... %>: Execução de lógica (sem saída).
    • <%* ... %>: Execução dinâmica (Dynamic commands).
    • <% ... %> (com tR ou retorno): Interpolação de saída.
  4. Execução: O código JS é executado de forma assíncrona (async/await é suportado e recomendado).
  5. Renderização: O resultado substitui os placeholders no buffer do editor ou no novo arquivo.

1.3 Modelo de Execução de Scripts

  • User Scripts: Arquivos .js localizados na pasta configurada pelo usuário. São carregados como módulos CommonJS (module.exports).
  • System Commands: Execução de comandos de shell do sistema operacional via child_process.
  • Acesso à API: Scripts têm acesso total ao app do Obsidian via tp.obsidian ou app.

2. Manual de Referência Técnica (Objeto tp)

O objeto tp é a porta de entrada para todas as funcionalidades.

2.1 Módulo tp.file (Manipulação de Arquivos)

Controla o arquivo atual ou cria novos arquivos.

Função Assinatura Descrição
create_new create_new(template, filename?, open?, folder?) Cria novo arquivo baseado em template ou string.
exists exists(filepath) Retorna boolean se o arquivo existe.
find_tfile find_tfile(filename) Retorna o objeto TFile do Obsidian.
move move(new_path) Move o arquivo atual para novo caminho.
rename rename(new_name) Renomeia o arquivo atual.
cursor cursor(order?) Define onde o cursor ficará após renderização.
cursor_append cursor_append(content) Adiciona texto após o cursor ativo.
include include(template) Inclui o conteúdo de outro template.

Exemplo Avançado (Criação Dinâmica):

1
// Cria uma nota em uma pasta baseada no ano atual e move o foco para ela
2
const folder = `Diário/${tp.date.now("YYYY")}`;
3
await tp.file.create_new("Template Base", "Nota do Dia", true, folder);

2.2 Módulo tp.system (Interação com Usuário)

Permite inputs durante a execução do template.

Função Assinatura Descrição
prompt prompt(text, default?, throw?, multiline?) Caixa de entrada de texto.
suggester suggester(text_items, items, throw?, place?, limit?) Menu de seleção única.
multi_suggester multi_suggester(...) Menu de seleção múltipla.
clipboard clipboard() Obtém conteúdo da área de transferência.

Exemplo Avançado (Seleção de Tags):

1
const tags = Object.keys(app.metadataCache.getTags());
2
const choice = await tp.system.suggester(tags, tags);
3
if (choice) {
4
    // Lógica com a tag escolhida
5
}

2.3 Módulo tp.date (Datas)

Manipulação de datas com base na biblioteca moment.js.

  • tp.date.now(format?, offset?, reference?, reference_format?)
  • tp.date.tomorrow(format?)
  • tp.date.yesterday(format?)
  • tp.date.weekday(format?, weekday_number, reference?, reference_format?)

2.4 Módulo tp.web & tp.obsidian (Integração Externa e Interna)

  • tp.web.daily_quote(): Retorna citação do dia.
  • tp.web.random_picture(...): Retorna imagem aleatória.
  • tp.obsidian: Expõe toda a API do Obsidian.
    • Útil para: tp.obsidian.requestUrl (HTTP requests), tp.obsidian.TFile, tp.obsidian.Notice.

2.5 Módulo tp.hooks

Ganchos de ciclo de vida.

  • on_all_templates_executed(callback): Executa algo após todos os templates serem processados.

3. Capacidades de Scripting (User Scripts)

3.1 Estrutura de User Scripts

Scripts devem ser arquivos .js na pasta configurada em Settings.

Padrão de Exportação:

1
// Arquivo: my_script.js
2
module.exports = async (tp) => {
3
    // Lógica complexa aqui
4
    const data = await fetch_data();
5
    return `Resultado: ${data}`;
6
}

Chamada no Template:
<% tp.user.my_script(tp) %> (Note a passagem de tp como argumento se necessário).

3.2 System Commands (Shell)

Definidos nas configurações, permitem execução de Bash/PowerShell.
Chamada: <% tp.user.nome_do_comando() %>.

4. Guia de Melhores Práticas

4.1 Otimização e Performance

  • Assincronicidade: Sempre use await para chamadas de I/O (tp.file, tp.system, tp.obsidian.requestUrl). Se esquecer o await, o template pode renderizar [object Promise].
  • Dynamic Command (<%*): Prefira <%* para blocos de lógica extensos que não devem produzir saída direta de texto, usando tR += "texto" para controlar a saída manualmente se necessário.
  • Cache: Evite ler o mesmo arquivo múltiplas vezes; armazene em variáveis.

4.2 Tratamento de Erros

  • Envolva chamadas externas (API, File System) em blocos try/catch.
  • Use new Notice("Erro...") via tp.obsidian.Notice para feedback visual ao usuário sem sujar a nota.

Exemplo Seguro:

1
<%*
2
try {
3
    const response = await tp.obsidian.requestUrl("https://api.exemplo.com/dados");
4
    tR += response.json.valor;
5
} catch (e) {
6
    new tp.obsidian.Notice("Falha na API: " + e.message);
7
    tR += "Dados indisponíveis";
8
}
9
%>
